## 📖 Overview

This project implements a production-grade deep learning system for **automated classification of rice leaf diseases**. It compares four state-of-the-art CNN architectures using transfer learning, and deploys the best-performing model as a free, publicly accessible web application.

**Detectable Diseases:**
| Disease | Pathogen | Visual Symptoms |
|---------|----------|-----------------|
| 🔴 Bacterial Blight | *Xanthomonas oryzae* | Yellow, water-soaked lesions; wilting |
| 🟠 Blast | *Magnaporthe oryzae* | Diamond-shaped lesions; gray centers |
| 🟤 Brown Spot | *Bipolaris oryzae* | Small dark brown circular spots |
| 🟡 Tungro | RTBV + RTSV (viral) | Yellow-orange leaves; stunted growth |

**Dataset:** 5,932 expert-annotated RGB images (source: Sethy et al., 2020)

## 🚀 Quick Start

### 1. Clone the Repository
```bash
git clone https://github.com/your-username/rice-disease-classifier.git
cd rice-disease-classifier
```

### 2. Create a Virtual Environment
```bash
# Windows
python -m venv venv
venv\Scripts\activate

# macOS / Linux
python -m venv venv
source venv/bin/activate
```

### 3. Install Dependencies
```bash
pip install -r requirements.txt
```

### 4. Set Up Your Dataset
Place your dataset in the following structure:
```
data/raw/Rice Leaf Disease Images/
    Bacterialblight/   (images...)
    Blast/             (images...)
    Brownspot/         (images...)
    Tungro/            (images...)
```

### 5. Preprocess Data
```bash
# Removes duplicates, splits into train/val/test
python src/data/preprocessor.py
```

### 6. Train All Models
```bash
# Train all 4 models with cross-validation
python src/models/train.py

# Train only VGG16
python src/models/train.py --model vgg16

# Skip cross-validation (faster)
python src/models/train.py --model vgg16 --skip-cv
```

### 7. Run the Web Application
```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```
Open your browser: **http://localhost:8000**

## 🧠 Model Architecture

### Transfer Learning Strategy

All models follow the same 3-stage approach:

```
Stage 1: Feature Extraction
  ┌─────────────────────────────┐
  │  Pre-trained Base (frozen)  │  ← ImageNet weights, all layers frozen
  │  e.g. VGG16 conv layers     │
  └────────────┬────────────────┘
               ↓
  ┌─────────────────────────────┐
  │  GlobalAveragePooling2D     │  ← Reduces feature maps to 1D
  │  Dense(1024, relu)          │  ← Task-specific learning
  │  Dropout(0.5)               │  ← Prevents overfitting
  │  Dense(4, softmax)          │  ← 4 disease classes
  └─────────────────────────────┘

Stage 2: Fine-tuning
  └── Unfreeze last 4 layers of base model
      → Gentle adaptation of pre-trained features
      → Very low learning rate (1e-5) to avoid destroying weights
```

### Why VGG16 Outperforms Others

| Reason | Explanation |
|--------|-------------|
| **Simplicity** | 3×3 conv filters throughout → stable gradient flow |
| **Depth vs. complexity** | Deep enough (16 layers) without inception/residual complexity |
| **Transfer learning efficiency** | ImageNet features generalize exceptionally well to leaf textures |
| **Training stability** | Less sensitive to learning rate than ResNet/Xception |
| **Statistical evidence** | McNemar's test confirms superiority (p < 0.05) |

## 📊 Statistical Validation

This project uses three statistical tests to rigorously compare models — a requirement for Master's-level research:

### 1. McNemar's Test (Test Set Comparison)
- Compares whether two models make **the same errors** on the same test samples
- Non-parametric, no distributional assumptions
- **Result:** VGG16 vs. all others: p < 0.05 (statistically significant)

### 2. Wilcoxon Signed-Rank Test (CV Fold Comparison)
- Compares model performance across **5 cross-validation folds**
- Better than t-test for small samples (5 folds)
- **Result:** VGG16 fold scores significantly higher than ResNet50, InceptionV3, Xception

### 3. Paired t-Test (Supplementary)
- Parametric comparison of mean fold accuracies
- Used alongside Wilcoxon for complete statistical picture

### 4. Bootstrap Confidence Intervals
- 95% CI computed via 1,000 bootstrap resamples
- Provides uncertainty bounds on all reported metrics

## 📈 Data Augmentation

Applied **only to training data** (never validation/test):

| Technique | Value | Purpose |
|-----------|-------|---------|
| Rotation | ±40° | Handles varying leaf orientations |
| Horizontal flip | Random | Doubles effective dataset size |
| Width/height shift | 20% | Simulates partial leaf views |
| Zoom | 20% | Handles varying distances |
| Shear | 0.2 | Simulates perspective distortion |
| Brightness | 0.5–1.5× | Handles different lighting |

## 🌐 API Documentation

Once running, visit **http://localhost:8000/docs** for interactive API docs.

### Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/` | Web UI |
| `GET` | `/health` | Server health check |
| `POST` | `/predict` | Predict disease from image |
| `POST` | `/predict-with-gradcam` | Predict + Grad-CAM explainability |
| `GET` | `/classes` | List all disease classes |

### Example API Request (Python)
```python
import requests

with open("rice_leaf.jpg", "rb") as f:
    response = requests.post(
        "http://localhost:8000/predict",
        files={"file": f}
    )

result = response.json()
print(f"Disease: {result['predicted_class']}")
print(f"Confidence: {result['confidence']:.1%}")
```
